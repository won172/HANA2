# HANA Budget Skill - UX Guidelines

## 문서 목적
이 문서는 관리자와 동아리 사용자의 화면을 수정할 때 따라야 할 UX 기준을 정리한다.

이 프로젝트는 예쁜 마케팅 페이지가 아니라  
**운영형 예산 플랫폼**이다.

---

## 가장 중요한 원칙

### 1. 관리자와 동아리의 목적은 다르다
관리자는 “통제와 검토”가 목적이다.  
동아리 사용자는 “신청과 집행”이 목적이다.

### 2. 사용자는 사후 설명보다 사전 이해가 필요하다
정책은 숨기지 말고 먼저 보여준다.

예:
- 허용 카테고리
- 금지 키워드
- 자동 승인 한도
- 수동 검토 기준
- 만료일

### 3. POS는 보조 도구다
POS 화면은 데모용이다.  
메인 흐름은 club/admin 안에서 완성되어야 한다.

---

## 관리자 UX 기준
관리자 홈에서 먼저 보여야 할 것:

1. 검토 대기 예산 신청
2. 보류 거래
3. 정산 대기
4. 만료 임박 예산
5. 최근 감사 이벤트
6. 전체 요약 수치

---

## 동아리 UX 기준
동아리 홈에서 먼저 보여야 할 것:

1. 남은 예산
2. 어떤 예산이 지금 쓸 수 있는지
3. 어떤 정책이 적용되는지
4. 보류/반려 건이 있는지
5. 정산해야 하는 예산이 있는지

예산 상세에서 반드시 있어야 할 것:
- 현재 잔액
- 사용 가능 기간
- 허용 카테고리
- 금지 카테고리
- 금지 키워드
- 자동 승인 한도
- 검토 기준
- 같은 화면에서 집행 요청

---

## 사유 문구 가이드
좋은 사유는:
- 짧다
- 사용자 행동이 이어진다
- 내부 코드명이 아니다

좋은 예:
- 신규 가맹점이어서 관리자 검토가 필요합니다.
- 금지 키워드가 포함되어 요청이 반려되었습니다.
- 자동 승인 한도를 초과해 승인 후 관리자에게 알림이 전달됩니다.
- 예산 사용 기한이 지나 요청할 수 없습니다.
